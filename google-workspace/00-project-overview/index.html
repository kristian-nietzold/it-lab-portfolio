Project Overview

Introduction

This project demonstrates the design and implementation of a production-style Google Workspace environment for a fictional organisation, Nietz Tech Ltd.

The lab replicates real-world administrative responsibilities performed by a Google Workspace Administrator, including user lifecycle management, organisational structuring, security configuration, and policy enforcement.

Objectives

  • Configure and verify a custom domain within Google Workspace
  • Design a scalable organisational unit (OU) structure
  • Create and manage users and groups
  • Implement role-based access control (RBAC)
  • Configure core security settings (MFA, sharing controls)
  • Apply data protection policies (DLP, alerts, audit logging)
  • Demonstrate administrative best practices

Environment Overview

  • Organisation: Nietz Tech Ltd
  • Primary Domain: nietz.co.uk
  • Platform: Google Workspace
  • Admin Console: admin.google.com

The environment simulates a small-to-medium business with multiple departments, role-based access requirements, and security-focused configuration.

Design Principles

  • Least Privilege Access: Users are granted only the access required
  • Separation of Duties: Admin roles are isolated from standard users
  • Scalability: Structure supports business growth
  • Security First: Policies prioritise data protection and risk reduction
  • Centralised Control: OU-based inheritance with controlled overrides

Outcome

This project demonstrates the ability to design, deploy, and manage a Google Workspace environment using real-world administrative practices.

  • Configured a full Workspace environment from domain to security
  • Implemented structured user and access management
  • Applied enterprise-level security concepts
  • Documented configurations clearly for operational use

This lab serves as both a practical learning project and a portfolio demonstration of hands-on Google Workspace administration skills.
