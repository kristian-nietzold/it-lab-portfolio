Device Management

Securing and managing user devices within Google Workspace

Overview

This section demonstrates how device access is secured and managed within Google Workspace using Basic Mobile Device Management (MDM).

  • Enforcing security policies on mobile devices
  • Controlling access based on compliance requirements
  • Monitoring and managing enrolled devices
  • Validating policy enforcement through real-world testing
Key concept
Device security is enforced through a combination of identity (2SV) and device compliance.
Note
Basic MDM provides essential controls without requiring device agents, making it suitable for BYOD environments.

Device Management Configuration

Mobile device management was configured using universal settings at organisational level.

Basic mobile management configuration

Configuration

  • Management level: Basic (agentless)
  • Scope: Organisation-wide (Nietz Ltd)

Capabilities Enabled

  • Remote account wipe
  • Password enforcement
  • Device approval tracking
Why this matters
Provides baseline protection while keeping deployment simple and scalable.

Configuration Applied

Settings applied across the organisation

  • Mobile management enforced for all users
  • Devices must meet minimum security requirements
  • Administrative visibility enabled

Policy Enforcement (2-Step Verification)

Access blocked due to missing 2-Step Verification

Outcome

  • Access automatically blocked
  • User required to enable 2SV
  • Identity-based access control enforced
Important
Device access is tied to identity security — non-compliant users cannot access company data.

Device Enrolment

Device successfully enrolled after meeting requirements

  • Status: Approved
  • Ownership: User-owned (BYOD)
  • Active sync confirms connection
Why this matters
Ensures only compliant devices can access organisational resources.

Device Inspection

Detailed device information in Admin console

Available Information

  • Device ID and resource ID
  • Operating system (Android 14)
  • Ownership type
  • Sync timestamps
  • Management level

Administrative Actions

  • Block device access
  • Wipe account data
  • Remove device from management
  • View audit logs

Devices Overview

Centralised device management dashboard

  • Separation of mobile, endpoint, and ChromeOS devices
  • Managed vs unmanaged visibility
  • Centralised policy control

Design Rationale

  • Basic MDM reduces complexity while maintaining control
  • 2-Step Verification enforces strong identity security
  • BYOD support allows flexible working while maintaining governance

Validation

  • Non-compliant devices are blocked
  • Compliant devices enrol successfully
  • Device status updates in real time
  • Admin actions function correctly

Real-World Considerations

  • Advanced MDM required for company-owned devices
  • BYOD environments rely heavily on identity controls
  • Device lifecycle processes must include onboarding and offboarding
  • Policies should scale with organisational growth

Summary

  • Device management successfully implemented
  • Identity and device security integrated
  • Policies validated through real-world testing
  • Centralised administrative control established
This configuration provides a secure and scalable approach to managing user devices in Google Workspace.
