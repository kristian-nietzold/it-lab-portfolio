Gmail Security & Configuration

Configuration of email protection, user controls, and threat mitigation within Google Workspace Gmail.

Overview

This section demonstrates how Gmail was configured to provide secure email communication while protecting users from phishing, malware, and spam threats.

The configuration focuses on combining automated threat detection with controlled user functionality to balance security and usability.

Key concept:
Email is the most common attack vector — securing it is critical to organisational security.

Spam, Phishing & Malware Protection

Gmail advanced protection settings

Advanced Gmail protections were configured to detect and block malicious emails before they reach users.

Configuration

  • Enhanced pre-delivery message scanning: Enabled
  • Enhanced malware and phishing protection: Enabled
  • Security sandbox: Disabled
Why this matters:
Threats are stopped before users interact with them, reducing reliance on user awareness.

Security Impact

  • Detects phishing links using machine learning
  • Blocks malicious attachments
  • Reduces risk of credential harvesting attacks
Security sandbox was left disabled as it typically requires higher-tier licensing and is more relevant in enterprise environments.

User Settings

Gmail user configuration settings

User-level Gmail settings were configured to support business workflows while maintaining appropriate security controls.

Configuration

  • Mail delegation: Enabled
  • Sender visibility: Owner and delegate visible
  • Group delegation: Disabled
  • Gmail offline: Disabled
  • Confidential mode: Enabled
  • Smart features: Enabled
Why this matters:
Enables collaboration (shared inboxes) while maintaining accountability and preventing uncontrolled access.

Design Decisions

  • Delegation enabled for shared mailbox workflows
  • Offline mode disabled to reduce data exposure risk
  • Confidential mode enabled for sensitive communications

Phishing Detection Test

Simulated phishing email correctly detected as spam

A simulated phishing email was sent to test Gmail’s threat detection capabilities.

Test Scenario

  • Fake PayPal verification email
  • External sender address
  • Suspicious link included

Result

  • Email automatically classified as Spam
  • Marked as External
  • User warned before interaction
Outcome:
Gmail successfully identified and blocked a phishing attempt without user intervention.

Email Authentication (DKIM)

DKIM authentication was configured during domain setup (see Domain Configuration section).

  • Outgoing emails are cryptographically signed
  • Email integrity is verified by receiving servers
  • Improves deliverability and trust
DKIM works alongside SPF and DMARC to provide full email authentication.

Summary

  • Advanced spam and phishing protection enabled
  • Secure and controlled user settings configured
  • Real-world phishing test successfully detected
  • Email authentication implemented (DKIM)
This configuration provides a secure and reliable email environment aligned with real-world organisational requirements.
