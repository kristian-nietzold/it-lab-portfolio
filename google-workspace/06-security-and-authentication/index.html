Security & Authentication

Implementation of modern identity protection controls to secure user accounts and reduce the risk of unauthorised access.

Overview

This section demonstrates how core authentication and credential policies were implemented to strengthen account security within the organisation.

The focus was on aligning with modern security best practices while maintaining usability for end users.

Key concept:
Strong identity security is the first line of defence against account compromise.

2-Step Verification (2SV)

2-Step Verification enforced at organisational level

2-Step Verification (multi-factor authentication) was enabled and enforced to add an additional layer of protection beyond passwords.

Configuration

  • Enforcement: Enabled
  • Methods: Any except SMS/phone where possible
  • Trusted devices: Allowed
  • Enrolment period: 1 day
Why this matters:
Even if a password is compromised, attackers cannot access accounts without the second authentication factor.

Security Impact

  • Protects against phishing attacks
  • Prevents credential stuffing
  • Reduces risk of account takeover
SMS-based authentication was restricted where possible due to known weaknesses such as SIM swapping.

Password Policy

Password policy enforcing strong credential standards

A secure password policy was configured to ensure strong and resilient credentials.

Configuration

  • Minimum length: 12 characters
  • Strong password enforcement: Enabled
  • Password reuse: Disabled
  • Password expiry: Disabled
  • Enforce at next sign-in: Disabled
Modern best practice:
Longer passwords are more secure than frequently changing shorter passwords.

Security Rationale

  • Improves resistance to brute-force attacks
  • Prevents reuse of compromised credentials
  • Avoids weak user behaviours caused by forced expiry
Password expiry was disabled intentionally, as it often leads users to create weaker, predictable passwords.

Security Design Approach

  • Layered security (password + MFA)
  • Balance between security and usability
  • Alignment with industry standards (NIST guidance)
  • Reduction of high-risk authentication methods

Summary

  • 2-Step Verification enforced across the organisation
  • Weak authentication methods restricted
  • Strong password policy implemented
  • Modern security practices applied
This creates a strong foundation for protecting user identities and preventing unauthorised access.
