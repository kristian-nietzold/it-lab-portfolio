# 08. Drive & Sharing


\# 08. Drive \& Sharing



This section demonstrates the configuration of Google Drive and file sharing controls within the organisation.



The objective is to enable collaboration while maintaining control over access and data exposure.



\---



\### Drive Access \& Sharing



Drive sharing settings were configured to allow internal collaboration while controlling external access.



!\[Drive sharing settings](screenshots/08-01-drive-sharing-settings.png)



Configuration includes:

\- Internal sharing: \*\*Enabled\*\*

\- External sharing: \*\*Allowed (controlled)\*\*

\- Link sharing: \*\*Restricted to intended users\*\*



This ensures:

\- Secure collaboration within the organisation  

\- Controlled external sharing  

\- Reduced risk of accidental data exposure  



\---



\### Shared Drives



Shared Drives were created to support team-based collaboration.



!\[Shared drives](screenshots/08-02-shared-drives.png)



This enables:

\- Centralised file ownership  

\- Persistent access independent of individual users  

\- Better management of team resources  



\---



\### Permission Management



Access to files and Shared Drives was managed using groups.



!\[Drive permissions](screenshots/08-03-drive-permissions.png)



This approach:

\- Simplifies access control  

\- Ensures consistency across users  

\- Supports scalable permission management  



\---



\### Design Approach



The configuration follows:



\- Least privilege access  

\- Group-based permission control  

\- Separation of internal and external sharing  



This ensures security while maintaining usability.



\---



\### Summary



This section demonstrates:



\- Configuration of Drive sharing settings  

\- Implementation of Shared Drives  

\- Use of groups for permission management  

\- Application of secure collaboration principles  



These controls provide a balance between accessibility and data protection.
